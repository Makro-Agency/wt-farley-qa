Before you start

  • Test the live theme and the draft.Live is wt-farley/main; the working draft is wt-farley/main fixes. Preview the draft with ?preview_theme_id=156346646666 appended to any URL.
  • Hard refresh between checks.Use Cmd/Ctrl+Shift+R so you are not seeing a cached page, especially after any theme push.
  • Check desktop and mobile for each page.Layouts differ; the mobile section at the bottom lists the mobile-specific things to confirm.
  • Flag anything about product wording, prices, images, or specs separately.Those are catalog/data items covered in the last section, not theme bugs.

Header, Navigation & Search revised

Present on every page

  • Sticky header stays on scroll.Header and menu remain pinned to the top as you scroll down a long page.
  • Mega menu opens for each top category.Hover/click each nav item; sub-categories render and links resolve to the right collections.
  • Quick search dropdown.Typing shows two columns: Categories and Products, with compact product cards (image, SKU, title, price).
  • Enter opens the full search page.Pressing Enter in the search box navigates to /search?q=…, it does not just close the dropdown.
  • Lookup mode (part number).Switching to Lookup and entering a part number behaves as designed (auto-fill / hand-off). Note any part numbers that return nothing — see data section.
  • Cart count badge.Adding an item updates the header cart count without a full reload.

Home

wt-farley.myshopify.com/

  • All sections render top to bottom.Banners, featured collections/products, and any promotional rows load with no broken images or empty blocks.
  • Links go to real destinations.Every button/tile links to an existing collection, product, or page (no 404s).
  • Featured product cards match the collection card design.Same layout, spacing, price, and Buy / Quick View button alignment as collection pages.

Collection page revised

/collections/all · and any category, e.g. /collections/cylinder-carts

Layout & cards

  • Toolbar spacing matches design.Padding above/below the toolbar and the gap down to the product grid look balanced (not cramped).
  • Product cards aligned.Image, title, one-line description, price, and Buy / Quick View buttons line up across the whole grid.
  • Card description shows one sentence.First sentence of the product description only; cards with no description hide the line cleanly (no empty gap).

Filters, sorting & loading

  • Sticky filters and toolbar.Filter sidebar and toolbar stay in view while the grid scrolls.
  • Filtering is instant.Selecting a checkbox filter updates the grid immediately, no visible "page stops then shows" jump.
  • Sorting works.Each sort option reorders the grid correctly.
  • Infinite scroll.Scrolling to the bottom loads more products automatically.
  • Price filter range.The "To" max should reflect the collection's real highest price. If it looks too low, that is a search-index / data item, see data section.

Search results page revised

/search?q=oxygen

  • Parity with the collection page.Product grid, cards, filters, sorting, list/grid toggle, sticky behaviour, and infinite scroll all behave the same as a collection page.
  • Relevant results.Searching a product name, SKU, or keyword returns sensible matches.
  • Grid and List modes.Both render like the collection page equivalents.
  • Empty query / no results.A search with no matches shows a clean "no results" state, not a broken layout.

Product page — standard new dropdowns

/products/oxygen-mini-body-click-regulator (multi-option example)

Custom variant dropdowns

  • Dropdowns use the new pill design.Each variant option shows a rounded pill with the selected value and a chevron, not the old grey box.
  • Opening shows the floating list.Clicking opens a white rounded panel of options; a second click / outside click / Esc closes it.
  • Selecting updates the product.Choosing a value updates price, SKU, availability, image, and the URL, exactly as before.
  • Out-of-stock options are greyed.Unavailable combinations show the option at reduced opacity and cannot be selected.
  • Colour options stay swatches.Colour-type options still render as swatches, not dropdowns.
  • Keyboard access.Tab to the dropdown, open with Enter/Space, arrow keys move through options, Enter selects.

Buy & media

  • Add to Cart.Adds the correct variant; cart count and drawer update.
  • Sticky Add-to-Cart bar.Appears on scroll and shows a single, correct price (no doubled price).
  • Specs / description render cleanly.No raw HTML tags or code visible in the feature bullets or description. Report any that appear (data item).

Product Configurator revised

/products/oxygen-hose?gas=selected

Image layering

  • Base assembly shows.Selecting a gas type shows the base hose in the preview.
  • Fittings layer onto the ends.Choosing an Inlet / Outlet fitting stacks its image at the correct end of the hose, aligned (not floating or centered).
  • Changing an option swaps its layer.Each new selection replaces the previous layer for that option immediately.
  • "Not Included" / blank options clear their layer.No leftover image remains when an option is set to none.
  • Options without artwork still work.Length, swivel, elbow, etc. with no layer image do not break the preview; the assembly still shows.

Part number, price & buy

  • Part number builds live.The Part Number field updates as options change.
  • Price updates live.Price reflects the base plus any option deltas.
  • Gas type switcher.Switching gas (e.g. Oxygen to Air) swaps to the correct sibling product / colour.
  • Reset configurator.Reset returns all options to defaults and clears the preview layers.
  • Add configured item to cart.The built assembly adds with the correct part number and price.
  • Part-number Lookup bar.Entering a known part number auto-fills the configurator. Note any that fail — see data section.

/cart and the cart drawer

  • Line items correct.Product title, selected options / configured part number, price, and quantity are right.

Data-migration issues (out of scope for this revision)

These surface on the storefront but originate from the migrated catalog data (Shift4Shop → Shopify) or from search indexing, not from the theme build. Listed here so they can be flagged and handled as a separate content/data task.

IssueWhere it showsWhy it is data, not theme
Empty product descriptions ~24 products (mostly gas hoses, some poles/racks) show a title but no description on the PDP and a blank description line on cards. These products only ever held the old "call 843-536-9050" shipping notice, which was removed as wrong/legacy. They need real copy written — a content task, not a template fix.
Understated price-filter maximum Some collections show a price-filter "To" value far below the actual highest product price. The theme reads Shopify's own filter range; the low number is a stale Search & Discovery index after migration + bulk edits. Resolves on reindex / re-saving the Price filter in the Search & Discovery app.
Sparse or missing specifications PDP feature/spec bullets are thin or absent on some products. Spec content was cleaned of raw HTML, but where the source data had little/no spec content there is nothing to show. Needs data entry per product.
Note: the Trustpilot testimonials widget depending on the Trustpilot account plan/allowed-domains is a third-party account setting, related but separate from catalog migration.
